# 不動産オーナー財務みえる化アプリ

決算書の大きな数字から、利益・手残り・借入の安全性を見える化し、不動産オーナーが「攻めるべきか、守るべきか」を考えるための簡易アプリです。

## 目的

このアプリは、税務申告や融資審査のための厳密な計算ツールではありません。

不動産オーナーが、決算書を見て「利益が出ているから安心」と考えるだけでなく、

- 実際にお金が残っているか
- 借入金が重すぎないか
- 将来修繕に備えても手残りがあるか
- 追加投資をしてよいか
- まず守りを優先すべきか

を確認するための入口ツールです。

## 画面構成

1. 決算書の数字を入れる
2. お金の残り方を見る
3. 借入の安全性を見る
4. 物件ごとに見る
5. 結果を印刷する
6. 次に見る資料

## 主な計算式

### 概算キャッシュフロー

```text
概算キャッシュフロー
＝ 当期利益
＋ 減価償却費
－ 借入元金返済額
－ 未払金・長期未払金の返済額
```

### 修繕積立後の手残り

```text
修繕積立後の手残り
＝ 概算キャッシュフロー
－ 将来修繕の年間積立目安
```

### 借入の安全性

```text
収益基準の借入上限 ＝ 年間賃貸収益 × 5
利益基準の借入上限 ＝ 営業利益 ÷ 10%
CF基準の借入上限 ＝ 返済原資CF × 15年

攻めの上限 ＝ 上記3つのうち一番低い金額
標準の目安 ＝ 攻めの上限 × 85%
守りの目安 ＝ 攻めの上限 × 70%
```

## 使い方

1. このフォルダをデスクトップに置く
2. `index.html` をブラウザで開く
3. 「サンプル入力」を押して動作を確認する
4. 数字を入れ替えて診断する
5. 「結果を印刷する」画面でA4レポートを印刷する

## GitHub Pagesで公開する場合

このアプリは静的ファイルだけで動きます。

```text
index.html
style.css
app.js
```

GitHubにアップロードし、GitHub Pagesを有効にすれば公開できます。

## 守秘義務・セキュリティ上の注意

- 関与先名、物件名、住所、個人名は入力しない運用を前提にしています。
- 入力データはサーバー送信していません。
- GitHub Pagesで公開する場合は、サンプルデータのみを置き、実データを保存しないでください。
- 実務利用する場合は、必ず税理士・専門家が内容を確認してください。
