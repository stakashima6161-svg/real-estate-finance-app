# Claude Code 用プロンプト

以下の方針で、現在の静的HTMLアプリを改善してください。

## 前提

不動産オーナー向けの「財務みえる化アプリ」です。

目的は、決算書の大きな数字から、

- 利益が出ているか
- 実際にお金が残っているか
- 借入が重すぎないか
- 攻めるべきか、守るべきか
- 必要に応じて物件別に原因を確認する

ことを、難しすぎない表現で見える化することです。

## 重要方針

- 専門用語を出しすぎない
- 不動産オーナーが直感的に理解できる画面にする
- 税務申告アプリではなく、経営判断の入口ツールとして作る
- 関与先名、物件名、住所、個人名を入れない設計にする
- 入力データはサーバー送信しない
- GitHub Pagesで公開できる静的構成にする

## 既存ファイル

- index.html
- style.css
- app.js
- README.md
- docs/specification.md

## 改善してほしいこと

1. UIをさらに見やすくする
2. スマホでも見やすくする
3. A4印刷レポートの見た目を整える
4. 計算エラーが出ないようにする
5. 入力項目の説明文をわかりやすくする
6. 診断コメントを自然な日本語にする
7. 物件別CFの入力行を増やせるようにする
8. 将来的にCSV出力やPDF出力しやすい構造にする

## 絶対に避けること

- 実データを保存する仕組みを勝手に入れない
- 外部APIに送信しない
- ログイン機能を入れない
- 難しい財務指標を前面に出しすぎない
- 関与先名や物件住所の入力欄を作らない

## 画面名

次の画面名は維持してください。

1. 決算書の数字を入れる
2. お金の残り方を見る
3. 借入の安全性を見る
4. 物件ごとに見る
5. 結果を印刷する
6. 次に見る資料

## 計算式

### 概算キャッシュフロー

当期利益 + 減価償却費 - 借入元金返済額 - 未払金返済額

### 修繕積立後の手残り

概算キャッシュフロー - 将来修繕の年間積立目安

### 借入上限

- 収益基準：年間賃貸収益 × 5
- 利益基準：営業利益 ÷ 10%
- CF基準：返済原資CF × 15

上記3つのうち一番低い金額を「攻めの上限」とする。

- 守りの目安：攻めの上限 × 70%
- 標準の目安：攻めの上限 × 85%
- 攻めの上限：攻めの上限 × 100%

## 最終的に目指すもの

不動産オーナーが、決算書を見ながら、

「利益は出ているけど、本当にお金は残っているのか」
「この借入水準で次の投資をしていいのか」
「今は攻める時期か、守る時期か」
「どの物件から確認すべきか」

を考えられるアプリにしてください。
